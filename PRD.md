# 产品需求文档 (PRD) - 智能机器人探索项目 (AI Robot Explorer)

## 1. 项目概述
### 1.1 项目背景
本项目是一个探索性的 AI 机器人交互应用，旨在通过整合视觉、听觉和先进的语言模型（Gemini），打造一个具有“生命感”和“个性化”的智能伴侣。

### 1.2 项目目标
- 实现多模态感知（视觉识别、声音检测、动作捕捉）。
- 建立基于行为树（Behavior Tree）的高级决策系统。
- 提供极致的用户体验，使机器人交互自然、不琐碎且具有预见性。

---

## 2. 核心功能需求

### 2.1 行为管理系统 (Behavior Manager)
**需求描述**：机器人需要一套鲁棒的优先级和冷却机制，避免“碎嘴”现象。
- **决策架构**：参考 `BehaviorTree.cpp` 的逻辑，实现基于状态和优先级的任务调度。
- **优先级设定**：
    - **P0 (最高)**：用户直接对话/指令。
    - **P1**：识别到特定人物（如：张三来了）。
    - **P2**：异常环境事件（如：巨大的响声、特殊的音色）。
    - **P3**：环境动态变化（如：检测到物体移动）。
    - **P10 (最低)**：闲时随机行为/自言自语。
- **冷却机制 (Cooldown)**：
    - 全局冷却：防止短时间内连续触发多次对话。
    - 针对性冷却：对同一人物或同一类事件设定独立的静默期。

### 2.2 视觉感知系统
- **人脸识别与注册**：支持用户上传照片或实时捕捉进行人脸注册，记录姓名和背景信息。
- **动作检测 (Motion Detection)**：实时分析摄像头画面，捕捉环境中的物体移动。
- **手势/姿态识别**：利用 MediaPipe 实现基础的人机交互感应。

### 2.3 听觉与语音系统
- **语音识别 (ASR)**：实时监听用户指令。
- **环境音分析**：检测大声喧哗或特定频率的声音。
- **语音合成 (TTS)**：使用 Gemini 2.5 Flash TTS 提供自然、有情感的语音反馈。

### 2.4 交互与情感表达
- **3D 机器人形象**：动态的 3D 模型，根据情绪（开心、思考、惊讶等）改变眼睛颜色和动作。
- **主动交互**：机器人不仅是应答工具，更会根据记忆（Firebase 存储）和环境变化主动发起话题。

## 3. 技术架构

### 3.1 核心技术栈
- **前端**：React + Vite + Tailwind CSS + Framer Motion.
- **3D 渲染**：Three.js (@react-three/fiber).
- **AI 模型**：Google Gemini 3.1 Pro (推理) / 3.0 Flash (实时交互).
- **感知库**：MediaPipe, TensorFlow.js, face-api.js.
- **后端/存储**：Firebase Auth & Firestore.

### 3.2 关键类设计
- `BehaviorManager`: 负责所有事件的入队、优先级过滤和执行。
- `RobotCharacter`: 负责 3D 模型的渲染与动画控制。
- `VisionService`: 封装摄像头流处理与视觉算法。

---

## 4. UI/UX 设计规范
- **布局**：采用可调节面板（Resizable Panels），左侧为 3D 机器人与监控视窗，右侧为对话历史与记忆库。
- **氛围**：深色调科技感，配合流光边框和毛玻璃效果。
- **反馈**：所有感知事件在 UI 上应有微弱的视觉反馈（如：检测到移动时边框闪烁）。
